Create a detailed project plan for **$ARGUMENTS**.

Read `AGENTS.md` and `README.md` first to understand the project architecture, then:

1. **Scope Analysis** — Understand the feature, dependencies, and integrations
2. **Task Breakdown** — Break down into testable units (migrations, models, services, controllers, frontend, tests)
3. **Effort Estimation** — Estimate each task (1h, 2h, 4h, 8h, 1d, 2d, etc.) and total effort
4. **Timeline Calculation** — Apply 15-20% buffer and estimate delivery dates
5. **Dependency Mapping** — Identify critical path, blockers, and parallel work
6. **Risk Assessment** — Call out unknowns, high-risk items, and mitigation
7. **Roadmap Creation** — Phase breakdown with deliverables for each phase

Provide output in structured markdown:
- Scope Summary
- Requirements Checklist
- Task Breakdown (organized by layer)
- Timeline Estimate
- Dependency Graph
- Risk & Mitigation
- Phased Roadmap
- Resource Requirements
